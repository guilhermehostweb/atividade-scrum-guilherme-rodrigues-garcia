# atividade-scrum-guilherme-rodrigues-garcia
Sistema simples de controle de tarefas.
Descrição do Sistema: Esse sistema de controle de tarefas permitira o cadastro do usuarios nesse sistema que criara uma lista aonde eles poderam adicionar qualquer tarefa que eles acharem necessario junto de poder separar essas tarefas nessa lista em partes como: a fazer, fazendo e concluida. Além de disso esse sistema permitira a esses usuarios o controle dessa lista e das demais tarefas nela junto do poder de organizalas do jeito que os usuarios acharem melhor e tambem permitira a edição dessas tarefas e o poder de visualizalas a qualquer momento.

Papeis do scrum:
Product Owner: Aquele que descubrira o que o cliente/usuario quer no seu sistema ou site.
Scrum Master: Aquele que ira liderar o time de desenvolvimento e lidera com possiveis obstaculos do time de desenvolvimento.
Time de Desenvolvimento: O grupo que ira desenvolver o sistema ou site.

Eventos do scrum:
Sprint: É Uma parte do metodo scrum que foco dividir a criação de um sistema ou site em partes chamadas sprints.
Sprint Planning: É o planejamento da splint aonde ocorre uma reunião aonde o foco é decidir o que será feito na splint.
Daily Scrum: É uma reunião rapido da equipe de desenvolvimento aonde eles alinham o trabalho em equipe e aonmde cada membro responde três perguntas: O que fez desde a última reunião?, O que vai fazer hoje? e Se existe algum impedimento?
Sprint Review: É uma reunião da equipe de desenvolvimento que ocorre no final de uma splint aonde eles monstram o que foi desenvolvida para os interessados no projeto que normalmente são os clientes.
Sprint Retrospective: É uma reunião feita ao fim de cada splint aonde ocorre uma avaliação de com a equipe foi durante o prcesso da splint.

Artefatos do scrum:
Product Backlog:O Product Backlog é uma lista de tudo que o sistema ou site que o cliente pediu precisa ter.
Sprint Backlog: É conjunto de tarefas e itens que a equipe que está usando o metodo scrum se compromete a desenvolver durante uma sprint.Ele é formado a partir do Product Backlog e detalha o que será feito e como será feito naquele período podendo sofrer reajustes.
Incremento: o incremento é a parte do produto que foi realmente concluida e está pronta para uso e entrega ao final da splint mesmo que o projeto ainda não esteja totalmente finalizado.

Product Backlog: lista de funcionalidades do sistema.
1: Cadastro dos usúarios no sistema de tarefas.
2: Capacidade de adiconar tarefas a lista do sistema.
3: Modificar essas tarefas adicionadas a lista além de poder apagar as feitas.
4: Historico de tarefas feitas.
5: Notificação e lembretes.

Priorização: Organizar por prioridade.

Burndown:
O que é um gráfico Burndown? ele é um grafico usado no metodo scrum que é nromalmente utilizado para ver e acompanhar o progresso de uma splint.
Para que ele serve no Scrum? para que a equipe saiba se está tendo progresso na splint.
